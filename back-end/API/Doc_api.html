Metadata

Up 
post /mashup
inserts a mashup (putMashup)
Inserts a mashup given a list of existing page ids. If one of the pages doesn't exist the document is not created

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body Mashup (optional)
Body Parameter —

Responses

201

mashup inserted

400

bad input parameter

404

one or more pages don't exist

500

internal error
Up 
get /search
returns a set of document (search)
Given a string (the search string) the server analyses it and returns a set of documents which better fit with the string

Query parameters

searchString (required)
Query Parameter — pass the string

Return type

DocumentsSet

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Set of document Ids and title. DocumentsSet

400

bad input parameter

500

internal error

Up 
get /document
retrieve a document (getDocument)
By passing in the appropriate ID, you can get the document you are looking for.

Query parameters

id (required)
Query Parameter — pass the id of the required document format: uuid

Return type

byte[]

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/pdf

Responses

200

stream the required pdf byte[]

400

bad input parameter

404

no doc with this id

500

internal error
